## How do I approach developing programs?
An important strategy that I learned woking on the 3D scene was that checking the results of my app changes each time a modification was made, no matter how small, will make sure I can see exactly what is going on in the scene. By making multiple changes, it was difficult to see what change in code reflected the change in the scene. Another strategy I used was to comment code that I did not understand in the beginning. As I made changes, I could see what my changes did, and then I made a comment to reflect what was changing so that I could go back to the code later without having to play a guessing game.

These stategies translated into iterative development. Making adjustments to one item at a time helped to make sure that it was less likely that I would have to go back and make changes to previously placed items to fit the scene. Also, by taking each step of the proces (object placement, texture, lighting) as its own iteration, there was less stress in making sure everything in the scene was cohesive.

I have always had a do everything at once and see if it works mentality. This course has taught me to slow down and take development in sections. By sectioning the work, the overall project became lest daunting and resulted in a better overall result.
